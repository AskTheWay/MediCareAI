## 🔧 核心功能详解 | Core Features

### 1. 混合检索 (Hybrid Search)

**实现位置 | Implementation:** `kb_vectorization_service.py:486-625`

**功能 | Features:**
- 结合向量相似度搜索 + PostgreSQL全文搜索 / Combines vector similarity + PostgreSQL full-text search
- 使用 RRF (Reciprocal Rank Fusion) 算法融合结果 / Uses RRF algorithm for result fusion
- 可配置的权重：vector_weight (默认0.7), keyword_weight (默认0.3) / Configurable weights

**API端点 | Endpoint:** `POST /api/v1/vector-embedding/knowledge-base/search`

**参数示例 | Parameters:**
```json
{
  "query": "糖尿病治疗方案",
  "disease_category": "endocrine",
  "source_type": "disease_guideline",
  "document_title": "糖尿病",
  "enable_hybrid": true,
  "vector_weight": 0.7,
  "keyword_weight": 0.3,
  "use_hyde": false,
  "top_k": 5
}
```

### 2. 元数据过滤 (Metadata Filtering)

**实现位置 | Implementation:** `kb_vectorization_service.py:626-717`

**支持的过滤器 | Supported Filters:**
- `disease_category`: 疾病分类 / Disease category (respiratory, cardiovascular, etc.)
- `source_type`: 来源类型 / Source type (disease_guideline, medical_document, research_paper, unified_kb)
- `document_title`: 文档标题（部分匹配）/ Document title (partial match)
- `section_title`: 章节标题（部分匹配）/ Section title (partial match)

### 3. HyDE 查询增强 (HyDE Query Enhancement)

**实现位置 | Implementation:** `rag_enhancement_service.py:40-144`

**功能 | Features:**
- 使用现有LLM生成假设性答案文档 / Generate hypothetical answer documents using existing LLM
- 查询改写：扩展缩写、添加同义词 / Query rewriting: expand abbreviations, add synonyms
- 提取5-8个相关医学关键词 / Extract 5-8 relevant medical keywords

**API端点 | Endpoint:** `POST /api/v1/vector-embedding/knowledge-base/enhance-query`

### 4. 上下文压缩 (Context Compression)

**实现位置 | Implementation:** `rag_enhancement_service.py:186-295`

**功能 | Features:**
- 使用LLM从检索结果中提取相关内容 / Extract relevant content from retrieved chunks using LLM
- 生成关键要点列表 / Generate key points list
- 保留源引用信息 / Preserve source reference information

**API端点 | Endpoint:** `POST /api/v1/vector-embedding/knowledge-base/compress`

## 📞 问题排查 | Troubleshooting

### 常见问题 | Common Issues

**Q: 全文搜索返回空结果？/ Full-text search returns empty results?**
A: 确保已运行数据库迁移，且 `search_vector` 列已填充。  
   Ensure database migration has been run and `search_vector` column is populated.

**Q: HyDE生成质量差？/ Poor HyDE generation quality?**
A: 检查AI模型配置是否正确，建议使用较强的模型（如GPT-4或GLM-4）。  
   Check AI model configuration, recommend stronger models (GPT-4 or GLM-4).

**Q: 检索速度变慢？/ Retrieval speed decreased?**
A: 可禁用混合搜索（`enable_hybrid: false`）或使用元数据过滤减少搜索范围。  
   Disable hybrid search or use metadata filtering to reduce search scope.
