## 1. 执行摘要 | Executive Summary

本文档详细分析了 MediCareAI 项目当前的 RAG（检索增强生成）架构实现，包括其设计特点、核心组件、技术流程，并提出了针对性的改进方案。

This document provides a detailed analysis of the current RAG (Retrieval-Augmented Generation) architecture implementation in the MediCareAI project, including its design characteristics, core components, technical workflows, and targeted improvement plans.

**核心发现 / Key Findings:**
- 当前架构采用**多路召回策略** (Multi-Path Retrieval)，结合疾病分类预过滤、向量检索和混合搜索
- 已实现 **HyDE** (Hypothetical Document Embeddings) 查询扩展和上下文压缩
- 知识库采用**扁平化设计**，所有文档统一存储在 `unified/` 目录
- 支持完整的**溯源追踪**，每个诊断结果都可追溯到具体的知识库来源

## 3. 核心组件详解 | Core Components in Detail

### 3.1 RAG 增强服务 (RAGEnhancementService)

**功能特点 / Features:**
- **HyDE (Hypothetical Document Embeddings)**: 生成假设性医疗文档用于更好的检索
- **查询改写 / Query Rewriting**: 扩展医学缩写、添加同义词、纠正拼写
- **上下文压缩 / Context Compression**: 从检索到的知识块中提取相关内容，减少噪声

**使用示例 / Example:**
```python
# HyDE 扩展示例
输入: "高血压吃什么药"
输出: {
    "hypothetical_document": "高血压患者药物治疗方案...",
    "expanded_keywords": ["高血压患者药物治疗方案", "降压药选择", "高血压用药指南"],
    "confidence": 0.95
}
```

### 3.2 多路召回 RAG 选择器 (MultiPathRAGSelector)

**多路召回策略 / Multi-Path Retrieval Strategy:**

| 路径 | 名称 | 策略 | 目的 |
|------|------|------|------|
| Path 1 | 分类过滤路径 | 疾病分类预测 → 分类过滤 → 向量检索 | 提高精度 |
| Path 2 | 全局路径 | 全库向量检索 | Fallback 保证召回率 |
| Path 3 | 关键词路径 | 医学术语精确匹配 | 捕捉特定术语 |
| Path 4 | 语义扩展路径 | 基于文档内容相关性重排序 | 提升相关性 |

**结果融合算法 (RRF):**
```python
score = sum(weight / (k + rank))
# 融合向量检索和关键词检索结果
```

### 3.3 知识库向量化服务 (KnowledgeBaseVectorizationService)

**文档处理流程 / Document Processing Flow:**
1. **Markdown 章节提取**: 从 Markdown 文档提取章节结构
2. **智能分块 / Smart Chunking**:
   - 分块大小: 1000 字符
   - 重叠: 200 字符
   - 分隔符优先级: `\n## ` → `\n### ` → `\n\n` → `\n` → `. ` → ` `
3. **去重 / Deduplication**: 使用 SHA-256 哈希检测重复内容
4. **嵌入生成 / Embedding Generation**: 批量调用嵌入模型 API
5. **存储 / Storage**: 存入 PostgreSQL 的 `knowledge_base_chunks` 表

**混合搜索 / Hybrid Search:**
```python
# 1. Vector Search (向量相似度搜索)
vector_results = await self._vector_search_with_filters(...)

# 2. Full-Text Search (PostgreSQL 全文搜索)
keyword_results = await self._fulltext_search_with_filters(...)

# 3. Reciprocal Rank Fusion (RRF 结果融合)
fused_results = self._reciprocal_rank_fusion(...)
```

### 3.4 数据模型设计 (Data Models)

#### `knowledge_base_chunks` - 知识库分块表

```sql
CREATE TABLE knowledge_base_chunks (
    id UUID PRIMARY KEY,

    chunk_text TEXT,
    chunk_text_hash VARCHAR(64),       -- 去重哈希
    embedding JSONB,                   -- 向量嵌入
    embedding_model_id VARCHAR(100),
    retrieval_count INTEGER DEFAULT 0,
    search_vector TSVECTOR,            -- PostgreSQL 全文搜索向量
    is_active BOOLEAN DEFAULT TRUE
);
```

### 6.1 短期改进 (1-2 个月) | Short-term Improvements

#### ✅ 改进 1: 添加重排序 (Re-ranking) 层 | COMPLETED [2026-03-01]

**状态 / Status**: ✅ 已完成 (Completed)

**问题**: 当前 RRF 融合只是简单的分数加权，没有真正理解查询和文档的深层相关性。

**实施方案 / Implementation**:

采用 **外部 API 重排序服务** 架构，支持多个国内云服务商的重排序 API：

```python
# 实现: backend/app/services/reranking_service.py
class RerankingService:
    async def rerank_chunks(self, query: str, chunks: List[Dict], top_n: int = 20):
        # 1. 从数据库加载配置
        config = await self._load_config()
        
        # 2. 获取对应提供商适配器
        adapter = self._get_adapter(config.provider)
        
        # 3. 调用外部 API 进行重排序
        results = await adapter.rerank(query, chunks, top_n)
        
        # 4. 缓存结果 (5分钟)
        await self._cache_result(cache_key, results)
        
        return results
```

**支持的提供商 / Supported Providers**:

| 提供商 | 名称 | 默认模型 | 特点 |
|--------|------|----------|------|
| **bailian** | 阿里云百炼 | gte-rerank | 国内稳定，适合医疗文档 |
| **bocha** | 博查 AI | bocha-rerank-v1 | 国内服务商 |
| **cohere** | Cohere | rerank-multilingual-v2.0 | 国际知名，多语言支持 |
| **jina** | Jina AI | jina-reranker-v2 | 高性能，长文本支持 |
| **custom** | 自定义 | 可配置 | 支持私有部署 |

**集成到 RAG 流程 / RAG Integration**:

在 `MultiPathRAGSelector` 的 Step 5.5 中集成：

```python
# backend/app/services/multi_path_rag_selector.py
async def _apply_external_reranking(self, query, chunks):
    # 转换 RetrievedChunk 为 dict 格式
    chunk_dicts = [...]

        query=query,

    return sorted_chunks
```

**配置方式 / Configuration**:

- **管理端配置**: 在 "AI模型配置" → "重排序模型 (Rerank)" 中配置
- **无硬编码**: 所有配置通过管理界面完成，无需修改代码或 .env
- **自动填充**: 选择提供商后自动填充默认 URL 和模型 ID
- **测试连通**: 支持一键测试重排序 API 连通性

**性能优化 / Performance Optimizations**:

- **缓存机制**: 重排序结果缓存 5 分钟，减少重复调用
- **批量处理**: 支持 Top-20 块重排序，控制 API 调用成本
- **超时控制**: 30 秒 API 调用超时，防止阻塞

**收益 / Benefits**: 提升检索精度 10-20%，无需本地 GPU 资源

## 7. BGE-Reranker 详细分析 | BGE-Reranker Detailed Analysis

### 7.1 模型概述 | Model Overview

BGE-Reranker 是由 **北京智源人工智能研究院 (BAAI)** 开发的开源重排序模型，专门用于提升 RAG 系统的检索精度。它采用 **Cross-Encoder** 架构，能够更准确地评估查询与文档之间的相关性。

The BGE-Reranker is an open-source reranking model developed by the **Beijing Academy of Artificial Intelligence (BAAI)**, specifically designed to improve retrieval accuracy in RAG systems. It uses a **Cross-Encoder** architecture to more accurately assess the relevance between queries and documents.

### 7.2 版本对比 | Version Comparison

| 模型 | 参数量 | 模型大小 | 显存占用 | 输入长度 | 多语言支持 |
|------|--------|----------|----------|----------|------------|
| **bge-reranker-base** | ~110M | ~400MB | ~0.9GB (FP16) | 512 tokens | 中英为主 |
| **bge-reranker-large** | ~340M | ~1.3GB | ~2.5GB (FP16) | 512 tokens | 中英为主 |
| **bge-reranker-v2-m3** | ~568M | ~2.2GB | ~2.1GB (FP16) | 8192 tokens | 100+ 语言 |

**推荐 / Recommendation**: 对于 MediCareAI 项目，建议使用 **bge-reranker-v2-m3**，原因：
- 支持 8192 token 长文本，适合医疗文档
- 100+ 语言支持，覆盖多语言场景
- 显存占用仅 2.1GB，部署成本低

```python
import torch

# 加载模型 / Load model
model_name = "BAAI/bge-reranker-v2-m3"
tokenizer = AutoTokenizer.from_pretrained(model_name)
model = AutoModelForSequenceClassification.from_pretrained(model_name)

# 启用半精度以加速推理 / Enable half precision
if torch.cuda.is_available():
    model = model.half().cuda()

model.eval()

def rerank(query: str, passages: list) -> list:
    """
    对查询与文档列表进行重排序
    Rerank query against list of passages
    """
    scores = []
    for passage in passages:
        inputs = tokenizer(
            query, passage,
            padding=True,
            truncation=True,
            return_tensors="pt",
            max_length=512
        )
        if torch.cuda.is_available():
            inputs = {k: v.cuda() for k, v in inputs.items()}
        
        with torch.no_grad():
            outputs = model(**inputs)
            score = outputs.logits.float().squeeze().item()
            scores.append(score)
    
    # 按分数降序排列 / Sort by score descending
    ranked_results = sorted(zip(scores, passages), reverse=True)
    return ranked_results

# 使用示例 / Usage example
query = "高血压患者的饮食注意事项"

    "高血压患者应减少钠盐摄入，每日不超过6克...",
    "糖尿病患者需要控制碳水化合物的摄入量...",
    "低盐低脂饮食有助于控制血压..."
]

results = rerank(query, passages)
```

```python
# backend/app/services/reranking_service.py

class BGERerankingService:
    """
    BGE-Reranker 重排序服务
    BGE-Reranker Reranking Service
    """
    
    def __init__(self, model_name: str = "BAAI/bge-reranker-v2-m3"):
        self.model_name = model_name
        self.tokenizer = None
        self.model = None
        self._load_model()
    
    def _load_model(self):
        """延迟加载模型 / Lazy load model"""
        if self.model is None:
            self.tokenizer = AutoTokenizer.from_pretrained(self.model_name)
            self.model = AutoModelForSequenceClassification.from_pretrained(
                self.model_name
            )
            if torch.cuda.is_available():
                self.model = self.model.half().cuda()
            self.model.eval()
    
    async def rerank_chunks(
        self, 
        query: str, 
        chunks: List[Dict], 
        top_k: int = 5
    ) -> List[Dict]:
        """
        对检索到的知识块进行重排序
        Rerank retrieved knowledge chunks
        
        Args:
            query: 用户查询 / User query
            chunks: 初步检索结果 / Initial retrieval results
            top_k: 返回前 K 个结果 / Return top K results
            
        Returns:
            重排序后的结果 / Reranked results
        """
        pairs = [[query, chunk['text']] for chunk in chunks]
        
        # 批量推理 / Batch inference
        scores = []
        batch_size = 8
        for i in range(0, len(pairs), batch_size):
            batch_pairs = pairs[i:i + batch_size]
            inputs = self.tokenizer(
                batch_pairs,
                padding=True,
                truncation=True,
                return_tensors="pt",
                max_length=512
            )
            
            if torch.cuda.is_available():
                inputs = {k: v.cuda() for k, v in inputs.items()}
            
            with torch.no_grad():
                outputs = self.model(**inputs)
                batch_scores = outputs.logits.float().squeeze().tolist()
                scores.extend(batch_scores if isinstance(batch_scores, list) else [batch_scores])
        
        # 更新分数并排序 / Update scores and sort
        for chunk, score in zip(chunks, scores):
            chunk['rerank_score'] = score
            chunk['final_score'] = 0.7 * score + 0.3 * chunk.get('similarity_score', 0)
        
        chunks.sort(key=lambda x: x['final_score'], reverse=True)
        return chunks[:top_k]
```
