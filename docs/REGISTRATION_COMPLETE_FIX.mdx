## 🐛 问题汇总 | Problem Summary

### 问题一：Android 注册字段丢失

**现象 | Symptoms:**
- Android App 注册时填写了所有字段（出生日期、性别、电话、地址、紧急联系人）
- 后端数据库只收到 email 和 full_name
- 其他字段全部为 null

**根本原因 | Root Cause:**
```kotlin
// MediCareApiClient.kt - 修复前
body?.let {
    setBody(it)  // ❌ 缺少 Content-Type 头部，JSON 序列化不完整
}
```

Ktor 的 `DefaultRequest` 插件设置了 Content-Type，但在 `setBody()` 时不生效，导致 Kotlinx Serialization 没有正确序列化所有字段。

**修复方案 | Fix:**
```kotlin
// 修复后
body?.let {
    contentType(ContentType.Application.Json)  // ✅ 显式设置 Content-Type
    setBody(it)
}
```

### 问题二：患者档案自动创建失败（500 错误）

**现象 | Symptoms:**
- 新注册用户登录后访问"个人中心"
- 页面显示"加载用户信息失败"
- 控制台报错：`GET /api/v1/patients/me 500 (Internal Server Error)`
- 错误信息：`'address' is an invalid keyword argument for Patient`

**根本原因 | Root Cause:**

1. **数据库模型变更未同步到服务层**
   ```python
   # Patient 模型（models.py）- 已移除 address 字段
   class Patient(Base):
       # ...
       # address 字段已删除 - 统一使用 User 表的 address
   
   # 但 patient_service.py 仍在使用
   db_patient = Patient(
       user_id=user_id,
       address=None,  # ❌ 无效的参数
       # ...
   )
   ```

2. **代码缩进错误**
   - 之前的修复尝试引入了重复代码块
   - 导致 Python `IndentationError`
   - 后端容器无法启动，返回 502 Bad Gateway

**修复方案 | Fix:**

1. 从 `create_patient()` 方法中移除 `address` 参数
2. 完全重写 `patient_service.py`，清理所有重复代码

---

### 问题三：注册时不创建患者档案

**现象 | Symptoms:**
- 用户注册成功
- 数据库 users 表有记录
- 但 patients 表没有对应记录
- 导致首次访问个人中心时需要"自动创建"

**修复方案 | Fix:**
修改 `auth.py` 注册端点，确保总是创建 `Patient` 档案：
```python
# 创建患者档案（总是创建，即使没有额外信息）
patient_data = PatientCreate(
    date_of_birth=date_of_birth,
    gender=user_data.gender,
    phone=user_data.phone,
    address=user_data.address,
    emergency_contact=emergency_contact,
)
await patient_service.create_patient(patient_data, user.id)
```

---

### 问题四：紧急联系人字段格式不一致

**现象 | Symptoms:**
- 有的 API 使用合并字段 `emergency_contact`
- 有的使用分离字段 `emergency_contact_name` + `emergency_contact_phone`
- Android 端和 Web 端显示不一致

**修复方案 | Fix:**
1. 数据库模型：保留分离字段
2. API Schema：添加分离字段到 `UserCreate`, `UserUpdate`, `UserResponse`
3. Android：更新 `RegisterRequest` 和 `Patient` 模型
4. 前端：更新 `User` 类型定义
