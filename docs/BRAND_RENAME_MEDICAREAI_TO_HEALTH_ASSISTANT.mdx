## 保留原因

当前没有直接修改所有 `MediCareAI` 出现位置，主要基于以下考虑：

- 邮件模板和邮件配置属于独立功能域，修改后需要额外验证邮件发送、模板渲染和管理员配置页展示。
- `.env`、`.env.example`、数据库模型默认值涉及运行配置和初始化默认值，修改前应确认是否需要保持兼容旧数据。
- `backend/app/main.py` 中的 API 标题、服务名和日志属于技术标识，不一定需要跟随用户端品牌一起替换。
- Docker 项目名、容器名、数据库名、包名、接口路径等技术标识暂不建议改动，避免影响启动、部署、数据卷、反向代理和外部调用。

## 后续全量替换建议

如果目标是全系统统一展示“健康助手”，建议按以下顺序继续：

1. 先替换患者端页面的用户可见文案。
2. 再替换管理员邮件配置页、后端邮件模板和邮件默认发件人名称。
3. 最后评估是否替换 API 标题、健康检查服务名、代码注释和文档中的 `MediCareAI`。
4. 替换后运行前端构建、后端启动检查，并至少验证登录页、患者首页、邮件配置测试和邮箱验证邮件。

## 验证方式

可使用以下命令检查剩余品牌文案：

```powershell
rg -n "MediCareAI|MedicareAI|健康助手" frontend backend .env .env.example docs -S
```

如只想检查仍未替换的旧品牌文案：

```powershell

```
