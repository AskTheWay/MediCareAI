## 🐛 问题描述 | Problem Description

### 现象 | Symptoms

医生端导出CSV文件时，检验数据（血常规、生化等指标）列为空，即使系统中已存在AI提取的检验数据。

When exporting CSV from the doctor portal, lab test columns (CBC, biochemistry, etc.) were empty, even though AI-extracted lab data existed in the system.

### 根本原因 | Root Cause

代码中存在**三套重复的数据提取逻辑**，导致AI提取的数据被后续的低质量提取覆盖。

There were **three sets of duplicate data extraction logic** in the code, causing AI-extracted data to be overwritten by subsequent low-quality extractions.

## 📞 问题排查 | Troubleshooting

### 常见问题 | Common Issues

**Q: CSV中某些检验值仍为空？**
A: 可能原因：
   - 原始检验报告图片质量差，AI无法提取
   - 该检验项不在提取模式列表中
   - 检查 `ExtractedLabReport` 表是否有数据

**Q: 如何验证数据提取是否正常？**
A: 检查后端日志：
   ```
   Case {case_id}: Found {n} AI-extracted reports with {m} items
   Case {case_id}: Final extraction has {k} lab values
   ```

**Q: 如何手动触发数据提取？**
A: 目前系统应在文档处理时自动提取，如需手动触发，可调用：
   ```python
   from app.services.ai_report_extractor import extract_lab_report_from_text
   result = await extract_lab_report_from_text(document_text, case_id, doc_id)
   ```
